CSS Cheat Sheet

CSS Selectors

Name HTML Tag CSS Example Description / Key Aspects
Type h1, p, etc. h1 { color: red } Selectors that come from the html tag. aka tag name or element selector
Universal * { color: red } Selects all elements of any type
Class class='brand' .brand { color: red } Selects any object with that class designation
Multiple Classes class='brand color' .brand { } .color { } Able to have two or more classes declared within one statement, must be seperated with a space
ID id='title' #title: { color: red } Unlike classes that can be used broadly throughout, an id can only have a single value and be used once
Attribute href [href] { color: red } [href] would target all elements with a href attribute
Granular Attribute img[src=....winter] img[src*='winter'] This would select an element with attribute of src that containts the string 'winter'
Pseudo Class p:hover p:visited p:hover { color: red} pseudo classes always start with a : and can be attached to any selector

Name CSS Example Description

Semantic HTML structure & Misc HTML

Tag Name Description
Semantic HTML is structured header -> nav -> main -> footer
<!DOCTYPE html> lets browser know what doctype
<html> contains all code
<head> contains the title of the page and other inf
<body> contains all the content
<header> usually for nav links or introductory content such as headings
<nav> for nav bar lists, can go in header or on its own
<main> the dominate content, between nav and footer
<footer> contains contact information etc.
<section> defines elements, such as chapters, headings, or any other area of the document with the same theme.
<article> holds content that makes sense on its own. can hold content such as articles, blogs etc.
<figure> encapsulates media like an image
<figcaption> adds a caption to a figure but is inside <figure> element
Name HTML Tag Description
Horizontal rule <hr> inserts a horizontal rule to seperate doc, no end tag
Break rule <br> inserts a line break which makes the text / imagine following moved to next line

The Box Model

Name CSS Example Description
width & height width: 20px; height: 1em; width and height of content area
padding padding: 16px; space between content area and border
border border: 3px solid coral; How thin or thick border is, what style of border, and color
border-radius border-radius: 5px; modifies the corner boxes of an element, can use 50% to get a perfect circle
padding padding: 10px; between contents of box and borders, can use padding-top: 10px; or padding: 10px 5px; (top&bot 10, right&left 5)
margin margin-bottom: 10px; the space directly outside of the box, remember margins stack horizontally not vertically
auto margin: 0 auto; centers an element in its containing element, but must have a width (width: 50px)
min / max width & height min width: 15px sets a min / max width for viewability
overflow overflow: hidden; controls what happens to contents that spills outside parent box, visible is default (hidden, scroll, visible)
resetting defaults * { margin: 0; padding: 0; } usually the first css rule in an external stylesheet, resets to default values
visibility visibility: hidden; changes the visibility (hidden, visible, collapse) hidden elements still reserve a space (unlike display: none;)
border-box * { box-sizing: border-box; } usually put at the top, makes the box width the actual width (as opposed to content-box)

Display & Position

Name CSS Example Description
static position: static; the default position
relative position: relative; used in conjunction with top: bottom: left: right: (remember it is offset and doesnt change position of other elements)
absolute position: absolute; will act as if no other elements are present
fixed position: fixed; will sty in a fixed place, like a navbar that stays at the top as the page moves
sticky position: sticky; will move with the screen until it hits a certain spot
z-index z-index: 3; determines which element shows on top, higher number will be shown first
inline display: inline an element will only take up as much space as is needed
block display: block; fill the entire width, but can set their width, they take up an entire line but only need the height necessary for their content
inline-block display: inline-block; combines both features of block and inline; can appear next to each other and specify their dimensions using width and height
float float: left; moves an element as far left or right as possible
clear clear: both; dictates how other elements behave and where they can touch, (both, left, right, none) both meaning neither side can touch another element, none means either side can

Flex box

Name CSS Example Description (red for declaration on flex containers only)
flex display: flex; changes an element to a block-level container with flex items inside
inline-flex display: inline-flex; allows multiple flex containers to appear inline with each other
justify content justify-content: flex-start; all items in order, starting from the left, no extra space between before or after
justify-content: flex-end; positioned in order with last item starting on right of container, no xtra space between before or after
justify-content: center; all items in order in the center of the container with no xtra space between before or after
justify-content: space-around; positioned with equal space before and after, resulting in double space between
justify-content: space-between; equal space between but no space before first or after last elements
align items align-items: flex-start; positioned at the top of parent container
align-items: flex-end; positioned at bottom of the parent container
align-items: center; center of all elements will be positioned halfway between top and bottom of parent
align-items: baseline; the bottom line of the conent will be aligned with each other (so moved to the point of the lowest item)
align-items: stretch; if possible items will stretch from top to bottom of a container (default value) elements with a specified height will not stretch, elements with a min height or no height will stretch
flex grow flex-grow: 1; determine by what rate an element will grow along the main axis (horizontal)
flex shrink flex-shrink: 1; same as above but with shrinking (default value is 1)
flex basis flex-basis: 100px determines the starting size of a flex element.
flex flex: 1 2 100px; shortened version: grow shrink basis
flex-wrap flex-wrap: wrap; child elements of a flex container that dont fit will go down a line
flex-wrap: wrap-reverse; same as wrap, but in a reverse order
flex-wrap: nowrap; stops items from wrapping, which is the default value
align content align-content: flex-start; all rows of elements will be positioned at the top of parent container
align-content: flex-end; all rows of elements will be positioned at bottom of the parent container
align-content: center; all rows of elements will be center of all elements will be positioned halfway between top and bottom of parent
align-content: space-between; all rows of elements will be spaced evenly from top the bottom with no space above first or below last
align-content: space-around; all rows of elements will be spaced evenly from top to the bottom of the container with same amount of space at top, bottom and between
align-content: stretch; if a min height or no height the rows of elements will stretch to fill the parent container from top to bottom (default)
flex-direction flex-direction: row; elements will be positioned from left to right across parent element (default)
flex-direction: row-reverse; same as wrap, but in a reverse order (right to left)
flex-direction: column; positioned from top to bottom starting from top left
flex-direction: column-reverse; positioned from bottom to top starting from bottom left
flex flow flew-flow: columnn wrap shorthand for flex-wrap & direction
